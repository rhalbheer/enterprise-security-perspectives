# Contributing to Enterprise Security Perspectives

Thank you for taking the time to contribute! This repository is intended to be an **open forum** for enterprise security thinking — a place where ideas can be exchanged, challenged, and refined. Security is not a solved problem, and no single perspective has all the answers. Your voice matters here, even — and especially — if it disagrees with what is already written.

---

## What This Repository Is About

This is a space for **driving security forward**: exploring how enterprise security should be designed, governed, and operated at scale. The goal is to advance thinking across the full breadth of the discipline — strategy, architecture, operations, governance, risk, and the human dimensions of security.

---

## What We Welcome

- **New perspectives and frameworks** — challenge existing ideas or propose entirely new ones
- **Corrections and improvements** — factual corrections, updated references, clearer explanations
- **Additional depth** — case studies, worked examples, or evidence that strengthens or refines an argument
- **Counter-arguments** — well-reasoned disagreement with content already in the repository is more than welcome; healthy debate moves the field forward
- **New topics** — relevant security subjects not yet covered here

> **Your contribution does not have to agree with what is already here.** Diverse and sometimes conflicting viewpoints are exactly what this repository needs.

---

## What We Do Not Accept

To keep this repository focused on advancing security thinking and free from commercial noise, the following types of contributions will not be accepted:

- **Product or vendor promotion** — content that endorses, advertises, or is primarily aimed at promoting a specific product, service, or vendor
- **Fundamentalist debates** — discussions framed as absolute, binary positions without nuance (e.g., *"Open Source vs. Closed Source"*, *"Cloud vs. on-prem"*, *"X is always right, Y is always wrong"*). Real-world enterprise security involves trade-offs; contributions should reflect that
- **Off-topic content** — material that is not related to enterprise security

---

## How to Contribute

### Raise an Issue First (Recommended)

Before investing time in a pull request, consider opening an [issue](https://github.com/rhalbheer/enterprise-security-perspectives/issues) to describe what you want to add or change. This allows for early discussion and avoids wasted effort.

Good issue titles are specific:
- ✅ *"Suggestion: add a section on operational technology (OT) security governance"*
- ✅ *"Counter-argument to the Zero Trust framing in the Comprehensive Edition"*
- ✅ *"Typo / factual error in Security at Machine Speed — Appendix"*

### Submitting a Pull Request

1. **Fork** the repository and create a new branch from `main`
2. Make your changes in Markdown (`.md` files)
3. Ensure your writing is clear, well-structured, and free from commercial promotion
4. Open a pull request with a concise description of what you changed and why
5. Be prepared to discuss and refine your contribution based on feedback

### Writing Style

- Write for an audience of **security leaders, architects, and practitioners** — people with real responsibility who need actionable thinking, not marketing
- Be clear and direct; avoid jargon where plain language works just as well
- Use evidence and reasoning to support your positions
- Acknowledge trade-offs honestly

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md). This community is built on respectful, evidence-based dialogue. Personal attacks, harassment, and bad-faith engagement have no place here.
