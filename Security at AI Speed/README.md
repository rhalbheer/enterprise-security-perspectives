## About This Section

This folder contains the **Security at AI Speed** series — a set of papers and frameworks for organizations that must now govern, operate, and lead in an environment where attackers leverage AI to compress the time between access and impact.

**The foundation of this series is *Your Security Model Is Too Slow to Matter*.** It delivers the leadership reality check that every other paper in this folder builds on. Start there. The follow-up papers — on governance and CISO leadership — are actionable companions that translate the core argument into specific operating models and hiring criteria.

## Core Themes

All papers in this section are grounded in the same set of strategic themes:

- **Decision latency as primary risk** — why the speed of seeing, deciding, and acting now determines outcomes more than control coverage
- **Pre-authored authority** — moving human authority forward in time so automated response can execute without waiting for approval
- **Threat-led governance** — replacing periodic review with a continuous operating loop that converts threat intelligence into enforceable decisions
- **AI-speed attacks** — how adversarial use of AI compresses reconnaissance, compromise, movement, and impact into a single operating flow
- **Identity as the control plane** — why standing privilege is a defect and AI agents must be governed as digital actors with lifecycle and scope controls
- **CISO as decision engineer** — the shift from running a security program to operating a decision system under time pressure
