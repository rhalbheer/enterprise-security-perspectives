# Security Policy

## About This Repository

This repository contains **thought leadership content** — long-form papers, role-tailored briefs, and practical frameworks on enterprise cybersecurity. There is no executable code. The "security" concerns here are therefore about **content integrity and responsible information quality**, not software vulnerabilities.

---

## What to Report

Because this is a content repository, the relevant issues to report are:

### 1. Factually Incorrect or Dangerously Misleading Guidance

If you identify content that contains **factual errors or outdated claims that could lead an organization to make a security decision that exposes it to harm**, please report it privately before raising a public issue. Examples:

- A recommended configuration, architecture pattern, or control that is technically incorrect and would weaken security if followed
- References to standards, regulations, or frameworks (e.g., NIS2, DORA, GDPR, EU AI Act) that are materially misrepresented in ways that could cause compliance failures
- Claims about attacker techniques, tools, or timelines that are demonstrably wrong and could lead to under-investment in a real risk

Minor factual corrections, stylistic disagreements, or differences of professional opinion do not need to be reported privately — open a public [issue](https://github.com/rhalbheer/enterprise-security-perspectives/issues) or pull request instead.

### 2. Content Integrity Concerns

If you believe content in this repository has been **tampered with, injected with misinformation, or altered in a way that misrepresents the author's intent and could mislead readers on security matters**, report this privately. This includes:

- Unauthorized modifications to documents that introduce incorrect or harmful security guidance
- Embedded links or references that point to malicious or misleading external resources

### 3. Out of Scope

The following are **not** security issues for this repository and should be handled through normal [issues](https://github.com/rhalbheer/enterprise-security-perspectives/issues) or pull requests:

- Typos, grammatical errors, or formatting issues
- Disagreements with the strategic direction or recommendations in the content
- Requests for new topics or additional content
- General feedback or questions about the material

## How to Report

To report a content integrity or potentially harmful factual issue privately, please **do not open a public issue**. Instead, contact the author directly:

- **Email:** roger@halbheer.ch
- **LinkedIn:** [linkedin.com/in/rhalbheer](https://www.linkedin.com/in/rhalbheer/)

Please include:
- The document name and section where the issue appears
- A clear description of the problem
- Why you believe it could lead to harm if followed
- Where possible, a reference or source that supports the correction

---

## Response Process

1. **Acknowledgement** — Reports will be acknowledged within **5 business days**.
2. **Assessment** — The report will be reviewed to determine whether it represents a genuine content risk or a matter of professional disagreement.
3. **Correction** — If the issue is confirmed, the affected document will be updated with a note in its changelog or version history. Significant corrections will be called out clearly so that readers who have already read the prior version are aware.
4. **Credit** — Reporters who identify genuine content issues will be credited in the document's acknowledgements unless they prefer to remain anonymous.

---

## Content Versioning and Freshness

All documents in this repository reflect the author's best understanding at the **time of publication**. The cybersecurity landscape evolves rapidly; guidance that was accurate at publication may require revision as threats, regulations, and technology change.

- Documents include a version or date indicator where relevant.
- If you are using content from this repository to inform decisions, always check whether a newer version is available.
- If you identify content that has become materially outdated in a way that poses a risk, please report it using the process above.

## Attribution and Integrity

All content is licensed under [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/). If you redistribute or adapt this material, you must:

- Attribute the original author clearly.
- Indicate if changes were made.
- Not present adapted content in a way that misrepresents the original author's views on security.
